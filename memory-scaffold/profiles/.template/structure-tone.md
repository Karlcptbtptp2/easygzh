# 主号结构习惯

视觉调性由主题 CSS 锁定（确定性）；结构习惯是 AI 协作时的参照（半结构化）。

## 开头破题

- 常用：一句话点出痛点 / 抛一个问题 / 直接给结论
- 避免：冗长寒暄、"今天给大家分享…"这类套话
- 典型开头长度：1-3 句

## 小标题命名

- 风格：陈述句或动宾短语，如"先定义问题"、"三个常见误区"
- 不用：纯数字编号（一、二、三）作唯一标题；不用 emoji 开头
- 层级：主要用 H2，偶尔 H3；几乎不用 H4

## 段落节奏

- 短段落为主，每段 2-4 句，避免大段文字墙
- 适当用列表、引用块打破节奏

## 结尾固定模块

- 标准结尾：一句话总结 → 一个开放式提问引导留言
- 可选模块："往期推荐"（仅当有相关历史文章时）
- 不加：求转发求关注的硬广话术（保持克制）

## 互动

- 文中偶尔用加粗/引用块强调关键句
- 不滥用分隔线，一篇文章 1-2 条为宜
